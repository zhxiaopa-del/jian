# 酒店查询

本技能用于在用户询问酒店、订房、住宿时，按规范收集需求并给出查询结果或指引。

## 何时使用

- 用户问「XX 有什么酒店」「XX 住哪里」「订房」「住宿推荐」
- 用户问「XX 酒店价格」「入住退房」「几号到几号有房」
- 用户需要某城市/景区的住宿建议或比价

## 需要收集的输入

在给出酒店建议前，尽量确认（用户未说时主动问）：

| 输入 | 说明 | 示例 |
|------|------|------|
| 城市/地点 | 目的地 | 北京、上海、杭州西湖 |
| 入住日期 | 入住日 | 2025-02-01、明天 |
| 退房日期 | 退房日 | 2025-02-03、住两晚 |
| 人数/房型 | 可选 | 2 人、大床房、家庭房 |

## 查询方式

1. **使用网络搜索**：根据「城市 + 入住退房日期 + 酒店」等关键词进行搜索，整理出酒店名称、区域、大致价格区间、评分（若有），并注明来源与「价格仅供参考、以平台为准」。
2. **平台指引**：若无法直接查到实时价格，给出可自行查询的平台与链接建议（见 [reference.md](reference.md)），并说明「实时价格与房态请以平台为准」。
3. **脚本（可选）**：在项目根目录执行可生成查询摘要与携程链接，供用户自行打开比价：
   ```bash
   python3 .cursor/skills/hotel-query/scripts/search_hotels.py <城市> [入住日期] [退房日期]
   ```
   示例：`python3 .cursor/skills/hotel-query/scripts/search_hotels.py 北京 2025-02-01 2025-02-03`

回复中应包含：目的地、日期范围、若干酒店或区域建议、价格区间（若搜到）、以及「建议在携程/美团/Booking 等平台核对实时价格与房态」。

## 回复示例结构

```
【目的地】北京
【日期】2025-02-01 至 2025-02-03（2 晚）
【建议】
- 区域：王府井/国贸/三里屯 等（按需求推荐）
- 酒店示例：XXX 酒店，约 XXX 元/晚起（来源：搜索/平台）
- 更多实时价格与房态请查询：携程 / 美团 / Booking（见 reference）
【说明】价格与房态以各平台为准，建议比价后下单。
```

若用户只问「北京住哪」，先简要推荐区域与 1～2 个酒店示例，再询问是否需要具体日期与价格查询。
