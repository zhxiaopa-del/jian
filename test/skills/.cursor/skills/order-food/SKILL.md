# 订餐助手

本技能用于在用户询问订餐、点餐、外卖、餐厅推荐时，按规范收集需求并给出建议或平台指引。

## 何时使用

- 用户问「吃什么」「附近有什么好吃的」「推荐一家餐厅」
- 用户问「点外卖」「订餐」「XX 附近外卖」「送餐」
- 用户问「XX 菜系」「辣的不辣的」「适合 X 人的餐厅」「预算 XXX 左右」
- 用户需要订座、点菜、团购、外卖比价

## 需要收集的输入

在给出订餐建议前，尽量确认（用户未说时主动问）：

| 输入 | 说明 | 示例 |
|------|------|------|
| 地点/区域 | 当前或目标位置 | 北京国贸、上海浦东、公司附近 |
| 口味/菜系 | 可选 | 川菜、粤菜、清淡、辣、西餐 |
| 人数 | 可选 | 1 人、2 人、聚餐 5 人 |
| 预算 | 可选 | 人均 50、100 以内、随便 |
| 忌口/偏好 | 可选 | 不吃辣、素食、不要香菜 |

## 查询方式

1. **使用网络搜索**：根据「地点 + 口味/菜系 + 餐厅/外卖」等关键词搜索，整理出餐厅名称、类型、人均、评分（若有），并注明「价格与营业状态以平台为准」。
2. **平台指引**：给出可自行点餐/订座的平台与说明（见 [reference.md](reference.md)），如美团外卖、饿了么、大众点评，并说明「下单与配送请以各 App 为准」。
3. **脚本（可选）**：在项目根目录执行可生成查询摘要与平台链接：
   ```bash
   python3 .cursor/skills/order-food/scripts/search_food.py <地点> [口味/菜系]
   ```
   示例：`python3 .cursor/skills/order-food/scripts/search_food.py 北京国贸 川菜`

回复中应包含：地点、口味/类型、若干餐厅或外卖建议、人均区间（若搜到）、以及「点餐/订座请使用美团/饿了么/大众点评等平台」。

## 回复示例结构

```
【订餐建议】
- 地点：北京国贸
- 类型：川菜 / 外卖
【推荐】
- XXX 餐厅，人均约 XX 元，评分 X.X（来源：搜索）
- 更多餐厅与实时价格请打开：美团外卖 / 饿了么 / 大众点评（见 reference）
【说明】营业时间与配送范围以各平台为准，建议在 App 内下单。
```

若用户只问「吃什么」，可先按地点或口味给 1～2 个建议，再问是否需要具体餐厅或外卖链接。
